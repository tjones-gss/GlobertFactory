How the codebase is laid out

Three clean layers. New features almost always slot into one of them — keep the simulation, the data/UI, and the rendering separate.

1 · Simulation

sim.js

The headless engine: tile grid, belt flow, recipes, producers, save format. No DOM. This is the source of truth — extend recipes, machines and economy here.

2 · Data & UI

erp.js · levels.js

Work orders, KPIs, office dashboard, career progression, the promotion overlay. Reads the sim, renders HTML panels, owns progression rules.

3 · Render & shell

floor.js · main.js · home.js

Canvas renderer + input, the game loop, the start screen. Phase 5's juice (FX, audio) plugs in here without touching the sim.
